HTTP Status Codes — Complete Reference Guide

Every HTTP status code explained with use cases, common causes, and troubleshooting tips.

What Are HTTP Status Codes?

HTTP status codes are three-digit numbers returned by a server in response to a client's request. They indicate whether the request was successful, redirected, had a client error, or encountered a server problem. Understanding these codes is essential for debugging APIs, troubleshooting web applications, and building reliable services.

Status codes are grouped into five classes based on the first digit:

ClassRangeMeaning
1xx100-199Informational — request received, continuing process
2xx200-299Success — request successfully received and processed
3xx300-399Redirection — further action needed to complete request
4xx400-499Client Error — request contains bad syntax or cannot be fulfilled
5xx500-599Server Error — server failed to fulfill a valid request

1xx — Informational

CodeNameWhen You See It
100ContinueServer received request headers and client should send the body. Used for large uploads where the client checks if the server will accept before sending.
101Switching ProtocolsClient requested a protocol upgrade (e.g., HTTP to WebSocket) and the server agreed.
103Early HintsServer sends headers before final response. Used for preloading resources while the server prepares the full response.

2xx — Success

CodeNameWhen You See It
200OKThe standard success response. GET returns the resource, PUT updates it, DELETE confirms removal. The most common status code on the web.
201CreatedPOST request successfully created a new resource. Response should include a Location header pointing to the new resource.
202AcceptedRequest accepted for processing but not yet complete. Used for async operations like batch jobs or long-running tasks.
204No ContentSuccessful request with no response body. Common for DELETE operations and PUT updates where nothing needs to be returned.
206Partial ContentServer returned part of the resource. Used with Range headers for resumable downloads and video streaming.

3xx — Redirection

CodeNameWhen You See It
301Moved PermanentlyResource permanently moved to a new URL. Browsers cache this. Use for domain changes or URL restructures. SEO: link equity transfers to new URL.
302FoundTemporary redirect. Browsers don't cache. Use for A/B testing, temporary maintenance pages. SEO: link equity stays on original URL.
304Not ModifiedResource hasn't changed since last request (conditional GET with If-None-Match / If-Modified-Since). Saves bandwidth — server sends no body.
307Temporary RedirectLike 302 but guarantees the HTTP method doesn't change (POST stays POST). Preferred over 302 for technical correctness.
308Permanent RedirectLike 301 but guarantees the HTTP method doesn't change. Preferred for permanent redirects in modern APIs.

4xx — Client Errors

CodeNameWhen You See ItHow to Fix
400Bad RequestMalformed syntax, invalid JSON, missing required fieldsCheck request body format, validate JSON, ensure all required fields present
401UnauthorizedMissing or invalid authentication credentialsCheck your auth token, API key, or session. Re-authenticate if token expired.
403ForbiddenValid auth but insufficient permissionsCheck user roles/permissions. You're authenticated but not authorized for this resource.
404Not FoundResource doesn't exist or URL is wrongVerify the URL. Check if the resource was deleted or moved.
405Method Not AllowedUsing wrong HTTP method (e.g., POST on a GET-only endpoint)Check API docs for allowed methods on this endpoint.
408Request TimeoutServer didn't receive complete request in timeRetry the request. Check network connectivity.
409ConflictRequest conflicts with current state (duplicate, version mismatch)Check for duplicates. Use If-Match for optimistic concurrency.
422Unprocessable EntityValid syntax but semantic errors (validation failures)Check field values against business rules. Read error details in response body.
429Too Many RequestsRate limit exceededWait for Retry-After header time. Implement exponential backoff.

5xx — Server Errors

CodeNameWhen You See ItHow to Fix
500Internal Server ErrorUncaught exception, configuration error, database connection failureCheck server logs. The most generic error — could be anything on the server side.
502Bad GatewayReverse proxy (nginx, Cloudflare) received an invalid response from upstream serverUpstream server is crashing or returning garbage. Check app server health.
503Service UnavailableServer overloaded, maintenance mode, dependency unavailableTemporary — retry with backoff. Check if autoscaling is configured.
504Gateway TimeoutUpstream server didn't respond in timeIncrease proxy timeout or fix slow upstream. Database query taking too long is common.
