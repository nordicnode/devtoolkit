How to Validate JSON — A Complete Guide

Everything you need to know about validating JSON data, from syntax rules to programmatic validation.

What is JSON Validation?

JSON validation is the process of checking whether a string of text conforms to the JSON (JavaScript Object Notation) syntax specification. Valid JSON can be parsed into a native data structure by any compliant parser. Invalid JSON will throw a parse error.

Validation is critical when consuming APIs, reading configuration files, or processing user input — malformed JSON causes runtime errors, silent failures, and hard-to-debug issues.

JSON Syntax Rules

JSON has a strict syntax. Here are the rules that must be followed:

RuleCorrectIncorrect
Keys must be quoted"name": "Alice"name: "Alice"
Strings use double quotes"hello"'hello'
No trailing commas[1, 2, 3][1, 2, 3,]
Comments not allowedN/A (use JSONC or JSON5)// comment
Strings for keys and string values"count": 5count: 0x5
Proper escaping in strings"path\\to\\file""path\to\file"

Common JSON Validation Errors

These are the most frequent errors developers encounter when writing or generating JSON:

ErrorCauseFix
Unexpected tokenUnquoted key or single-quoted stringUse double quotes for all keys and string values
Unexpected trailing commaComma after the last item in an object or arrayRemove the trailing comma
Unexpected end of inputMissing closing bracket or braceMatch all opening and closing delimiters
Bad control characterUnescaped newline or tab in a stringEscape with \\n or \\t
Bad stringInvalid escape sequence like \\xOnly use valid escape sequences: \\\", \\\\, \\\/, \\b, \\f, \\n, \\r, \\t, \\uXXXX
