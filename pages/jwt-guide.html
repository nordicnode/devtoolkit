Understanding JWT Structure — A Developer's Guide

A practical deep-dive into how JSON Web Tokens work, what each part contains, and how to use them safely.

What is a JWT?

A JSON Web Token (JWT, pronounced "jot") is a compact, URL-safe way to represent claims between two parties. A JWT encodes a set of claims as a JSON object that is used as the payload of a JSON Web Signature (JWS) or JSON Web Encryption (JWE) structure.

JWTs are commonly used for authentication (proving identity) and authorization (proving permissions). They're sent in HTTP headers, stored in cookies, or passed as URL parameters.

A JWT looks like this: xxxxx.yyyyy.zzzzz — three Base64URL-encoded strings separated by dots.

The Three Parts

1. Header

The header typically consists of two parts: the token type (typ) and the signing algorithm (alg).

{
  "alg": "HS256",
  "typ": "JWT"
}

Common algorithms: HS256 (HMAC + SHA-256), RS256 (RSA + SHA-256), ES256 (ECDSA + SHA-256).

2. Payload

The payload contains the claims — statements about an entity (typically a user) and additional data. Claims come in three types:

TypeExamplesDescription
Registerediss, sub, aud, exp, iatPredefined claim names with specific semantics
Publicname, email, roleRegistered in the IANA JWT registry or collison-resistant
Privateapp-specific fieldsCustom claims agreed upon by the parties

3. Signature

The signature is used to verify the token hasn't been tampered with. It's computed from the encoded header, encoded payload, and a secret (for HMAC) or private key (for RSA/ECDSA):

HMACSHA256(base64UrlEncode(header) + "." + base64UrlEncode(payload), secret)

Security Considerations

  • Never store sensitive data in a JWT payload — it's Base64-encoded, not encrypted. Anyone can decode it.
  • Always verify the signature server-side — decoding alone doesn't prove authenticity.
  • Set short expiration times — use the exp claim. 15 minutes for access tokens, 7 days for refresh tokens.
  • Don't accept alg: none — this attack removes the signature. Your server must enforce the expected algorithm.
  • Use HTTPS — JWTs sent over HTTP can be intercepted.
  • Consider token revocation — JWTs are stateless by design. Implement a blacklist or short expiry if you need revocation.
